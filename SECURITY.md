# Security Policy

## Reporting a vulnerability

If you discover a security issue or potential attack vector within Nextly or any of its dependencies, please **do not** open a public GitHub issue. Report it privately by emailing [info@nextlyhq.com](mailto:info@nextlyhq.com) and we will respond as quickly as we can.

When reporting, please include a description of the issue, steps to reproduce, the affected package(s), and any known mitigations — this helps us triage and fix the issue faster.

You'll be credited in any advisory we publish unless you prefer to remain anonymous.
